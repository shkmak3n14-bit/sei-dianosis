# Core（自己理解モジュール）

UI より先に実装方針を共有するための記録です。
実装順序は **UI 先行**。本ドキュメントは Core に入る責務の合意事項です。

Core は UI 非依存のドメイン層です。画面表示は `ui/mobile/` 側が担当します。

---

## `/logic` — 自己理解モジュールのロジック

| 内容 | 説明 |
|------|------|
| 深掘りの進め方 | 自己理解セッションの進行ルール |
| 質問の分岐 | 回答に応じて次の質問を切り替える |
| ユーザーの回答に応じた補正 | 回答内容から解釈・重み付けを調整する |
| キャラの反応ルール | どのタイミングで、どう反応するかを決める |

---

## `/data` — 自己理解に必要なデータ

| 内容 | 説明 |
|------|------|
| タイプごとの説明 | 各タイプの自己理解向け解説 |
| 行動傾向 | 日常で出やすい行動パターン |
| ストレス時の反応 | 負荷がかかったときの典型反応 |
| 誤解ポイント | 自分／他者から誤解されやすい点 |
| 自己理解の進捗ログ | セッション進捗・回答履歴などの記録データ |

---

## `/character` — キャラクター定義

| 内容 | 説明 |
|------|------|
| キャラの人格モデル | キャラの基本性格・役割 |
| 口調 | 話し方・語尾・トーン |
| 表情 | 状況に応じた表情バリエーション |
| タイプごとの性格 | ユーザーのタイプに合わせたキャラ側の振る舞い差分 |
| 寄り添い方 | 共感・励まし・問いかけのスタイル |

---

## `/questions` — 自己理解モジュールの質問セット

| 内容 | 説明 |
|------|------|
| 深掘り質問 | 動機・感情・価値観を掘り下げる質問 |
| 行動パターン確認 | 日常の行動傾向を確認する質問 |
| 過去の経験を聞く質問 | 体験ベースで自己理解を促す質問 |
| 誤解ポイントの説明 | 誤解されやすい点を扱う問い／解説 |

---

## `/diagnosis_engine` — 診断結果の変換層

タイプ診断（`type-engine`）の結果を、自己理解モジュールが使える形に渡すための層です。

| 内容 | 説明 |
|------|------|
| タイプ判定結果を読み込む | 診断モジュールの出力を受け取る |
| 自己理解モジュール用に整形する | Core／UI が扱いやすいデータ形へ変換する |

> 診断ロジック自体は持たない。**読み取りと整形のみ**。

---

## 依存関係（想定）

```
type-engine（診断結果）
        ↓ 読み取り
diagnosis_engine（変換・整形）
        ↓
logic / questions / data / character
        ↓
ui/mobile（表示）
```
